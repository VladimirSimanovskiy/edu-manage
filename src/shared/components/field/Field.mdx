# Field

`Field` представляет собой универсальный контейнер для создания полей ввода с поддержкой меток, описаний и различных состояний. Позволяет создавать согласованные и доступные поля ввода с гибкой настройкой внешнего вида и поведения.

[Ссылка на Figma](https://www.figma.com/design/lc2aOtrukFLJQwxxgH3Cgm/UI-kit?node-id=18672-40037&p=f&t=fqPE6zhGSWCbrorh-0).

### Базовое использование

<Canvas of={ComponentStories.Default} />

```tsx
import { Field } from "ics-ui-kit/components/field";
import { User } from "lucide-react";
import { Icon } from "ics-ui-kit/components/icon";
import { TextInput } from "ics-ui-kit/components/input";

function MyField() {
	return (
		<Field
			title="Имя пользователя"
			description="Ваше публичное имя пользователя"
			required
			control={({ readonly, id }) => (
				<TextInput
					placeholder="Введите имя пользователя"
					startIcon={<Icon icon={User} />}
					id={id}
					readOnly={readonly}
				/>
			)}
		/>
	);
}
```

### Свойства

- `title` - заголовок поля

- `description` - описание поля

- `required` - обязательное поле

- `readonly` - режим только для чтения

- `descriptionPlacement` - расположение описания

    - `text` - текстовое описание
    - `icon` - описание в виде иконки с подсказкой

- `control` - функция рендеринга контрола поля

- `error` - текст ошибки

- `layout` - расположение элементов

    - `horizontal` - горизонтальное расположение
    - `vertical` - вертикальное расположение

- `className` - кастомные стили

- `labelClassName` - кастомные стили для лейбла

### Доступность

- Компонент автоматически генерирует уникальные `ID` для полей
- Для улучшения доступности рекомендуется:
    - Использовать описательные заголовки
    - Добавлять описания для сложных полей
    - Обеспечивать достаточный контраст для состояний
    - Использовать семантически правильные `HTML-элементы`

### Рекомендации

- Используйте обязательные поля только когда это действительно необходимо
- Добавляйте описания для сложных или важных полей
- Выбирайте подходящее расположение элементов в зависимости от контекста
- Используйте состояние ошибки для валидации
- Используйте `className` и `labelClassName` для тонкой настройки стилей

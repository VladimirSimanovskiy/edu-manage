# FormField

`FormField` - это компонент-надстройка над базовым компонентом `Field`, который добавляет адаптивный дизайн и дополнительные возможности для полей формы.

[Ссылка на Figma](https://www.figma.com/design/lc2aOtrukFLJQwxxgH3Cgm/UI-kit?node-id=18836-29081).

## Использование

- Основан на компоненте `Field`
- Поддерживает адаптивный дизайн
- Основные сценарии использования:
    - Создание адаптивных полей формы
    - Управление расположением элементов поля
    - Интеграция с различными компонентами ввода
    - Обеспечение консистентного отображения полей формы

### Базовое использование

<Canvas of={ComponentStories.Default} />

```tsx
import { FormField } from "ics-ui-kit/components/form";
import { TextInput } from "ics-ui-kit/components/input";
import { Icon } from "ics-ui-kit/components/icon";
import { User } from "lucide-react";

export const Basic = () => {
	return (
		<FormField
			name="username"
			title="Имя пользователя"
			control={({ field, readonly }) => (
				<div className="flex gap-2">
					<TextInput
						placeholder="Введите имя пользователя"
						startIcon={<Icon icon={User} />}
						readOnly={readonly}
						{...field}
					/>
					{!readonly && <IconButton variant="outline" icon={Search} onClick={form.handleSubmit(onSubmit)} />}
				</div>
			)}
		/>
	);
};
```

### Свойства

- `name` - имя поля формы

- `title` - заголовок поля (лейбл)

- `description` - описание поля

- `control` - функция рендеринга контрола

- `required` - является ли поле обязательным

- `layout` - вариант раскладки: `"responsive"` (по умолчанию), `"vertical"`, `"horizontal"`

- `className` - дополнительные классы

- `labelClassName` - классы для лейбла

- `readonly` - поле только для чтения

- оставшиеся свойства компонента `Field`

### Доступность

- Обеспечивает правильную фокусировку и навигацию с клавиатуры
- Поддерживает достаточный контраст для видимости
- Правильно обрабатывает состояния ошибок
- Обеспечивает правильную связь между лейблом и полем ввода

### Рекомендации

- Используйте для всех полей формы
- Добавляйте описания для сложных полей
- Указывайте `required` для обязательных полей
- Используйте `labelClassName` для настройки стилей лейбла
- Обеспечьте достаточный контраст для видимости
- Используйте в сочетании с другими компонентами формы
- Учитывайте адаптивное поведение при проектировании форм
